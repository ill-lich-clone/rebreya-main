Диагностика, выполненная перед правками:
- Проверены файлы UI глобальных ивентов:
  - scripts/ui/global-events-app.js
  - templates/global-events-app.hbs
  - styles/main.css
- Обнаружено:
  - В global-events-app.js функция getSelectionOptions брала данные только из repository.dataset/repository.model без принудительной загрузки модели, из-за чего при ранних состояниях UI списки могли быть пустыми.
  - В emptyEditorState стартовая дата была пустой строкой.
  - В styles/main.css не было достаточно жестких переопределений для select/option в темной теме Foundry; в результате текст в некоторых селектах/опциях был трудночитаемым.
  - В templates/global-events-app.hbs найдена битая кириллица (кракозябры вида "Р..."), что нарушает требование по UTF-8.

Принятые решения (архитектурно):
1) Сделать сбор опций асинхронным и надежным:
- Добавить в global-events-app.js асинхронную загрузку опций через await moduleApi.getModel() при необходимости.
- Собирать options не из одного источника, а объединять model + dataset с дедупликацией.
- Сортировать списки по localeCompare("ru") для удобства в UI.
- Формировать понятные подписи:
  - города: "Название (Государство)" при наличии государства,
  - регионы: "Название (Государство)" при наличии государства,
  - государства: из stateSummaries + из city/region state.

2) Ввести дефолт даты начала:
- Добавить resolveDefaultStartDate(moduleApi):
  - основной источник: moduleApi.getCalendarSnapshot().isoDate (текущая дата игрового календаря),
  - fallback: текущая UTC дата системы в формате YYYY-MM-DD.
- В openEditor:
  - стартовать state от emptyEditorState(defaultStartDate),
  - если создается новый ивент и startDate пустая -> выставлять defaultStartDate автоматически.

3) Починить контраст и читаемость полей:
- Добавить специализированные CSS-переопределения для .rm-global-event-editor:
  - input/select/textarea + select option с контрастным цветом текста и фона,
  - стили для placeholder,
  - отдельная читаемость select[multiple] и option:checked.

4) Убрать битую кириллицу:
- Полностью перезаписать templates/global-events-app.hbs в UTF-8 с корректными русскими строками.
- Обновлены заголовки, кнопки, статусы, пустое состояние.
